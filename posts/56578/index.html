一、作用

call、apply、bind作用是改变函数执行时的上下文，简而言之就是改变函数运行时的this指向的

二、区别

也就是说，区别是，当你希望改变上下文环境之后并非立即执行，而是回调执行的时候，使用 bind() 方法。而 apply/call 则会立即执行函数。

bind比较适合将this绑定后的函数传入到其他函数中去执行，特别是作为回掉函数异步执行。

再总结一下：

  • apply 、 call 、bind 三者都是用来改变函数的this对象的指向的
  • apply 、 call 、bind 三者第一个参数都是this要指向的对象，也就是想指定的上下文；如果没有这个参数或参数为undefined或null，则默认指向全局window
  • apply 、 call 、bind 三者都可以利用后续参数传参；
  • bind 是返回对应函数，便于稍后调用；apply 、call 则是立即调用

call 、bind 、 apply 这三个函数的第一个参数都是 this 的指向对象，第二个参数差别就来了：

  • call 的参数是直接放进去的，第二、第三、第 n 个参数全都用逗号分隔，直接放到后面 obj.myFun.call(targetObj,”args1”,”args2”,……)。
  • apply 的所有参数都必须放在一个 数组 里面传进去 obj.myFun.apply(targetObj,[‘args1’, …, ‘string’ ])。
  • bind 除了返回是函数以外，它的参数和 call 一样。
  • 当然，三者的参数不限定是 string 类型，允许是各种类型，包括函数、object 等等！

三、实现call、apply、bind函数

每个javascript函数其实都是Function对象，而Function对象是构造函数，构造函数是有原型对象的，也就是Function.prototype，这个属性里有很多我们可以使用的属性跟方法，比如call、apply、bind这些就是绑定在原型对象属性里面的，因此我们要模仿就必须在原型对象里面添加新的和call一样的属性。

apply 跟call方法一样，只不过参数有点区别

其他相关知识

第 1 题：call 和 apply 的区别是什么，哪个性能更好一些？

  1. Function.prototype.apply 和 Function.prototype.call 的作用是一样的，区 别在于传入参数的不同；
  2. 第一个参数都是，指定函数体内 this 的指向；
  3. 第二个参数开始不同，apply 是传入带下标的集合，数组或者类数组， apply 把它传给函数作为参数，call 从第二个开始传入的参数是不固定的，都会 传给函数作为参数。
  4. call 比 apply 的性能要好，平常可以多用 call, call 传入参数的格式正是内 部所需要的格式
