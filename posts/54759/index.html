1.简单的排序算法

简单的排序算法是深入学习其他算法的基础

1.1冒泡排序

冒泡排序算法相对于其它排序算法运行效率是最低的，但是在概念上它是排序算法中最简单的，是最适合刚开始学习排序算法的初学者。

冒泡排序的思路：

  • 对应未排序的数组各元素从头到尾 依次比较相邻的两个元素大小关系
  • 如果左边的数字大，则两个元素交换位置
  • 向右移动一个位置，比较下面两个元素的大小
  • 当比较到最右端的时候，最大的元素就被放在了最右变得位置
  • 按照这个思路，在最左端重新开始，这次只需要比较到倒数第二个位置的元素即可
  • 以此类推，就可以完成数据的排序

冒泡排序的过程：

  • 外层循环取变量 j，内层循环取变量 i；
  • 第一步：首先我们应该思考的是 外层循环 中数组每完成一轮比较，下次再进行比较的长度是依次减少的。而且比较交换的是i和i+1的位置，所以长度应该取数组的长度-1，防止取i+1越界。
  • 第二步：内层循环，我们只需要让 i<j ,因为外层循环中的j在不断减少，我们要比较的次数正好是到 j 的位置。

冒泡排序的效率：

比较次数：

  • 对于 N 项数据：（N-1)+(N-2)+…+2+1 = N * (N-1)/2(真正的比较次数)
  • 大O表示法：O(N2)
    交换次数：
  • 真实的次数：N*(N-1) / 4;
  • 比较两次才需要交换一次（不可能每次比较都需要交换），那么交换次数为N2/4;
  • 大O表示法：O(N2)

1.2选择排序

选择排序改进了冒泡排序

  • 将减缓次数有O(N2)减少到O(N)
  • 但是比较次数依然是O(N2)

选择排序的思路：

  • 主要思想就是：用一个变量min从头开始记录元素的索引
  • 首先选定记录 第一个元素索引的位置，然后取第一个元素的值和后面的元素 依次比较
  • 如果后面的元素的值 小于 记录索引的元素的值，那么min就等于 更小元素的索引
  • 经过一轮比较后，min记录的是最小元素的索引，就可以与第一个元素的位置进行交换，确定第一个位置是最小的
  • 然后使用同样的方法把剩下的元素逐个比较即可
  • 可以看出选择排序，第一轮会选出最小值，第二轮会选出第二小的值，直到最后

比较一次的代码逻辑：第一轮会选出第一个位置的最小值

选择排序的效率：

比较次数：

  • 对于 N 项数据：（N-1)+(N-2)+…+2+1 = N * (N-1)/2(真正的比较次数)
  • 选择排序和冒泡排序的比较次数真实次数：N * (N-1)/2
  • 大O表示法：O(N2)
    交换次数：
  • 选择排序的交换次数只有 N-1 次
  • 大O表示法：O(N)
  • 所以选择排序通常认为在执行效率上是高于冒泡排序的

1.3插入排序

插入排序是简单排序中效率最高的，也是学习其他高级排序算法的基础，比如 希尔排序、快速排序。

插入排序的思路：

  • 插入排序的核心是： 局部有序
  • 从第一个元素开始，该元素可以被认为 局部有序
  • 取下一个元素，在已经排序的元素序列中从后向前扫描
  • 如果 该元素 （已排序）大于 新元素，将该元素移到下一个位置上
  • 重复上一个步骤，直到找到已排序的元素中 小于或者等于 新元素的位置
  • 将该元素插入到该位置上，重复上面的步骤

插入排序的效率：

比较次数：

  • 最多次数：1+2+3+…+(N-1) = N*(N-1)/2
  • 然而每趟发现插入点之前，平均只有全体数据项的一半需要进行比较
  • 我们可以除以2 得到 N*(N-1)/4，所以相对于选择排序，比较次数是少了一半的
    插入的复制次数：
  • 复制次数：1+2+3+…+(N-1) = N*(N-1)/2
  • 复制效率消耗的性能是比较低的
  • 平均次数：N*(N-1)/4

1.4希尔排序 shell

  • 希尔排序 基于 插入排序，是插入排序的一种高效的改进版，并且效率比插入排序要更快
  • 希尔排序按其设计者 希尔（Donald Shell)的名字来命名，该算法由 1959年公布
  • 所有的限制都是从自己的内心开始的

希尔排序的思路：

希尔排序的增量：

  • 选择合适的增量

    • 在希尔排序的原稿中，他建议的 初始间距 是 N/2，简单的把每趟排序分成两半
    • 也就是说，对于 N = 100，增量间隔序列为：50，25，12，6，3，1
    • 这个方法的好处就是不需要在开始排序之前为找到合适的增量而进行任何的计算

  • Hibbard 增量序列：

    • 增量的算法： 2k - 1,也就是为1、3、5、7…等等
    • 这种增量的最坏复杂度为 O（N3/2),猜想的平均复杂度为O(N5/4),目前尚未证明

  • Sedgewick增量序列

    • {1,5,19,41,109,…},该序列中的项或者是 94i - 9*2i +1 或者是 4i - 32i +1
    • 这种增量的最坏复杂度为 O（N4/3),猜想的平均复杂度为O(N7/6),目前尚未证明

希尔排序的效率：

  • 希尔排序的效率跟增量有关系的
  • 最坏的情况下是：O(N2)，但是通常都要好于O(N2)

总之，我们使用希尔排序大多数情况下效率都高于简单排序

希尔排序举例说明：

  • 存在这样一个数组———>arr = [8,9,1,7,2,3,5,4,6,0];

  • 当gap=5时：交换之后的—>arr = [3,5,1,6,0,8,9,4,7,2];

  • 当gap=2时：交换之后的—>arr = [0,2,1,4,3,5,7,6,9,8];

  1. 获取数组的分量 gap = 10/2 = 5,也就是分为五个小组:[8,3],[9,5],[1,4],[7,6],[2,0]
  2. while循环中 对五组分别进行插入排序
    • 比较前 用临时变量temp，保存这个位置的信息
    • 当gap = 5时：分为5个小组:[8,3],[9,5],[1,4],[7,6],[2,0]
    • 当gap = 5时：取在gap处的第一个元素3，在内部while循环中和 8（j-gap=0）比较，8比3大，8移动到3的位置，j-=gap就是将 j 移动到8的位置，但是0>5 -1不成立，最后3到了8的位置
    • 当gap = 2时：分为2个小组:[3,1,0,9,7],[5,6,8,4,2]
    • 当gap = 2时：取在gap处的第一个元素1，依次往后循环，当前元素跟前面j-gap处的元素进行插入排序的步骤
    • 当gap = 1时：分为1组，元素所在的位置基本确定，只需要移动很小的位置就可以了

1.5快速排序

快速排序是目前所有排序算法中效率最高的、最快的一种排序算法

  • 快速排序也被列为20世纪十大算法之一

希尔排序相当于插入排序的升级版，快速排序其实就是我们学习过的最慢的冒泡排序的升级版

  • 我们知道冒泡排序需要经过跟多次交换，才能在一次循环中，将最大的值放在正确位置上
  • 而快速排序可以在一次循环中（其实是递归的调用），就可以找出某个元素的正确位置，并且之后该元素不需要任何移动了

快速排序最重要的思想是： 分而治之

快速排序的思路：

  • 选取所有元素中的一个中位数，按照大小交换三个元素的位置，然后将中位数跟倒数第二个元素交换位置，并且返回中位数的值（也就是枢纽函数的做法）
  • 定义两个变量，用来记录左、右两个指针，左指针在0处开始，右指针在right-1处开始
  • 当遇到左指针指向的元素大于枢纽值，左指针停下，右指针开始查找，当右指针找到 小于 枢纽值的元素，右指针停下，此时交换两个指针所在位置的两个元素
  • 依次查找，直到 左指针 大于等于 右指针时，结束此次递归，交换此时 左指针 所在位置的元素 与 枢纽值的位置
  • 分而治之，继续递归调用交换过后枢纽值所在位置的前后

快速排序的平均效率：

  • 平均效率是 O(N*logN);
  • 虽然某些其他排序算法也可以达到O(N*logN)，但是快速排序是最好的
